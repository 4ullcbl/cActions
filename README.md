CActions — это мощный инструмент, который позволяет легко добавлять интерактивность и настраиваемое поведение на ваш сервер с помощью простых текстовых команд в конфигурационных файлах. Если вы используете плагины, поддерживающие CActions, вы сможете описывать цепочки действий (сообщения, звуки, частицы, команды и многое другое) прямо в конфигах, без необходимости писать сложный код или устанавливать множество дополнительных плагинов.

## О системе📘
CActions работает как “движок” для обработки действий, записанных в удобном текстовом формате. Действия выглядят так:
```[имя-действия] аргумент1, аргумент2, ...```
Все аргументы разделяются запятой с пробелом, поэтому внутри аргументов можно использовать пробелы (например, текст сообщения).

Эти действия могут быть вписаны в любые секции конфигурационных файлов плагинов, которые интегрированы с CActions. При наступлении события (например, игрок прыгнул, зашёл на сервер, убил моба) плагин выполнит все прописанные действия — отправит сообщение, проиграет звук, вызовет частицы и т.д.

## Установка🔃
1. Скачайте последнюю версию CActions с официальной страницы.
2. Поместите файл CActions.jar в папку plugins вашего сервера.
3. Перезапустите сервер или выполните команду /reload confirm (для Paper / Spigot).
4. Убедитесь, что плагин загрузился без ошибок (в консоли должно быть сообщение о успешной загрузке).

После установки CActions будет автоматически доступен для всех поддерживаемых плагинов — они смогут использовать его функционал в своих конфигах.

## Возможности⚡
 
- Простой и понятный синтаксис — действия записываются в одну строку и легко читаются даже новичками.

- Встроенные действия (могут быть расширены другими плагинами):

  ```yaml
  [message] Текст сообщения # — отправляет сообщение игроку.
  [broadcast] Текст # — отправляет сообщение всем игрокам на сервере.
  [sound] ENTITY_PLAYER_LEVELUP, 1, 1 # — проигрывает звук (название, громкость, высота тона).
  [particle] FLAME, 10, 0, 0, 0, 0.2 # — создаёт частицы (тип, количество, смещения, скорость).
  [command] say Привет, мир! # — выполняет команду от имени консоли.
  [player_command] spawn # — выполняет команду от имени игрока.
  # И многие другие — список зависит от установленных плагинов-дополнений.
  ```
- Гибкость и расширяемость — разработчики плагинов могут легко добавлять свои собственные действия, которые сразу же станут доступны в конфигах.
- Поддержка значений по умолчанию — если аргумент не указан, можно задать значение, которое будет использоваться автоматически (например, цвет частиц или текст сообщения).
- Контекст выполнения — действия могут использовать информацию об игроке, местоположении и другие данные в момент выполнения (например, координаты игрока, его имя и т.д.).

## Примеры использования в конфиге

Допустим, у вас есть плагин, который вызывает действия при прыжке игрока. В его конфиге вы можете прописать:
yaml

```yaml
actions:
  - "[message] Ты прыгнул! Отличный прыжок!"
  - "[sound] ENTITY_EXPERIENCE_ORB_PICKUP, 1, 1.2"
  - "[particle] CLOUD, 20, 0.5, 0, 0.5, 0.05"
```
Теперь каждый раз, когда игрок прыгает, он будет видеть сообщение, слышать звук и видеть облако частиц вокруг себя.
А если вы используете плагин на убийство мобов, можно сделать так:

```yaml

on_mob_kill:
  - "[broadcast] &eИгрок %player% убил %mob%!"
  - "[command] give %player% diamond 1"
  - "[sound] ENTITY_PLAYER_LEVELUP, 1, 1"
```
В этом примере используются плейсхолдеры (%player%, %mob%), которые автоматически заменяются на реальные данные.
